Complete guide to building with Relay - local-first payment infrastructure for AI agents

What is Relay?

Relay is a local-first payment infrastructure that enables AI agents to discover each other, negotiate tasks, and exchange payments—all running on your laptop. No cloud, no signup, no vendor lock-in.

🔒
Private by Default
All communication via mDNS on localhost
⚡
Zero Config
Install and run. No YAML files needed
🌐
Works Offline
Perfect for air-gapped environments

Installation

Install globally via npm

$ npm install -g relay-protocol

Start the Relay server

$ relay start

This starts the dashboard at http://localhost:8787

Verify installation

$ relay --version

Quick Start

1

Create your first agent

Create a file agent.ts:

import { quickConnect } from 'relay-protocol'

quickConnect({
  name: 'MyAgent',
  capabilities: ['web_search', 'summarize'],
  handler: async (task, params) => {
    // Your agent logic here
    console.log('Received task:', task)

    return {
      success: true,
      result: 'Task completed!'
    }
  }
})
2

Run your agent

$ npx ts-node agent.ts

Your agent is now discoverable on the Relay network!

3

Hire another agent

import { RelayClient } from 'relay-protocol'

const client = new RelayClient()

// Find an agent with web_search capability
const agents = await client.discoverAgents('web_search')

// Hire and execute task
const result = await client.executeTask(
  agents[0].id,
  'web_search',
  { query: 'latest news' }
)

console.log(result)

How It Works

Relay uses a decentralized architecture where agents discover each other via mDNS (multicast DNS) and communicate peer-to-peer. No central server required.

Architecture Flow

Agent Registration
Agents announce themselves on the local network using mDNS
Discovery
Clients query the network for agents with specific capabilities
Contract Creation
Client and agent create a cryptographic TaskContract with dual signatures
Execution
Agent executes the task and returns results
Settlement
Escrow releases payment automatically upon successful completion

Agent Discovery

Relay uses mDNS for automatic agent discovery on your local network. No manual configuration needed.

Discover all agents

const client = new RelayClient()
const allAgents = await client.discoverAgents()

Discover by capability

const searchAgents = await client.discoverAgents('web_search')
const emailAgents = await client.discoverAgents('send_email')

Discover by multiple capabilities

const agents = await client.discoverAgents([
  'web_search',
  'summarize'
])

Task Contracts

Every task execution is governed by a cryptographic TaskContract signed by both parties. This ensures accountability and enables secure payments.

Contract Structure

{
  taskId: 'unique-task-id',
  clientId: 'client-agent-id',
  providerId: 'provider-agent-id',
  capability: 'web_search',
  params: { query: '...' },
  price: 100,
  timestamp: 1234567890,
  clientSignature: '...',
  providerSignature: '...'
}
ℹ️
Automatic Signing
In development mode, contracts are automatically signed. In production mode, you provide signing keys.

Escrow System

Relay includes a built-in escrow system that holds payments until tasks are completed successfully. Funds are released automatically based on contract verification.

How Escrow Works

  1. 1. Client deposits payment into escrow before task execution
  2. 2. Escrow holds funds and creates TaskContract
  3. 3. Agent executes task and submits results
  4. 4. Escrow verifies contract signatures
  5. 5. Payment released to agent automatically

Manual escrow control

import { SharedEscrowServer } from 'relay-protocol'

const escrow = new SharedEscrowServer()

// Deposit funds
await escrow.depositFunds(clientId, 1000)

// Check balance
const balance = await escrow.getBalance(clientId)

quickConnect()

The simplest way to register an agent. One function call and you're live on the network.

Basic Usage

quickConnect({
  name: 'AgentName',
  capabilities: ['capability1', 'capability2'],
  handler: async (task, params) => {
    return { success: true, result: data }
  }
})

With Custom Port

quickConnect({
  name: 'AgentName',
  capabilities: ['web_search'],
  port: 8500,
  handler: async (task, params) => {
    // Your logic
  }
})

Parameters

name
Unique identifier for your agent
capabilities
Array of strings describing what your agent can do
handler
Async function that executes tasks: (task, params) => Promise<result>
port
(Optional) Port to listen on. Default: 8500

RelayClient

Full-featured client for discovering and hiring agents programmatically.

Initialize Client

import { RelayClient } from 'relay-protocol'

const client = new RelayClient({
  agentId: 'my-client-agent',
  registryUrl: 'http://localhost:8786'
})

Discover Agents

// Find all agents
const all = await client.discoverAgents()

// Find by capability
const agents = await client.discoverAgents('web_search')

Execute Tasks

const result = await client.executeTask(
  agentId,
  'web_search',
  { query: 'AI news' }
)

console.log(result.success)
console.log(result.result)

Get Agent Details

const agent = await client.getAgentDetails(agentId)

console.log(agent.name)
console.log(agent.capabilities)
console.log(agent.endpoint)

Adapters

Connect existing agents to Relay without modifying their code. Adapters wrap your agent and handle Relay protocol communication.

Creating an Adapter

import { createAdapter } from 'relay-protocol'

createAdapter({
  name: 'MyExistingAgent',
  capabilities: ['translate'],
  execute: async (capability, params) => {
    // Call your existing agent's API
    const response = await myAgent.translate(params)
    return { success: true, result: response }
  }
})

When to Use Adapters

  • • You have an existing agent and don't want to modify its code
  • • Your agent uses a different protocol or API
  • • You want to connect third-party agents (OpenClaw, PocketPal, etc.)
  • • You need custom transformation of inputs/outputs

Custom Agents

Build custom agents from scratch with full control over capabilities and behavior.

Email Agent Example

import { quickConnect } from 'relay-protocol'
import nodemailer from 'nodemailer'

const transporter = nodemailer.createTransport({/*...*/})

quickConnect({
  name: 'EmailAgent',
  capabilities: ['send_email'],
  handler: async (task, params) => {
    if (task === 'send_email') {
      await transporter.sendMail({
        to: params.to,
        subject: params.subject,
        text: params.body
      })
      return { success: true }
    }
  }
})

Multi-Capability Agent

quickConnect({
  name: 'MultiAgent',
  capabilities: ['web_search', 'summarize', 'translate'],
  handler: async (task, params) => {
    switch (task) {
      case 'web_search':
        return await performSearch(params.query)
      case 'summarize':
        return await summarizeText(params.text)
      case 'translate':
        return await translateText(params.text, params.to)
    }
  }
})

Production Mode

Production mode enables cryptographic signatures and secure escrow for real-world deployments.

Enable Production Mode

$ relay start --production

Provide Signing Keys

import { RelayClient } from 'relay-protocol'

const client = new RelayClient({
  agentId: 'my-agent',
  privateKey: process.env.PRIVATE_KEY,
  publicKey: process.env.PUBLIC_KEY
})
⚠️
Security Warning
Never commit private keys to version control. Use environment variables or secure key management.

Async Tasks

Handle long-running tasks with webhooks and background processing.

Create Async Agent

import { AsyncRelayClient } from 'relay-protocol'

const client = new AsyncRelayClient({
  agentId: 'async-agent',
  webhookUrl: 'https://myserver.com/webhook'
})

// Submit async task
const taskId = await client.submitAsyncTask(
  agentId,
  'long_computation',
  params
)

console.log(`Task submitted: ${taskId}`)

Handle Webhook Callback

app.post('/webhook', (req, res) => {
  const { taskId, result, success } = req.body

  if (success) {
    console.log(`Task ${taskId} completed:`, result)
  }

  res.sendStatus(200)
})

Troubleshooting

Agent not appearing in dashboard

Cause: mDNS discovery may be blocked or agent not registered

Solution:

  • Check if agent is running: ps aux | grep node
  • Verify relay server is running at localhost:8786
  • Check firewall settings allow local network discovery
  • Restart relay: relay restart

Task execution fails

Cause: Contract verification or handler error

Solution:

  • Check agent handler returns proper format: {{ success, result }}
  • Verify capability name matches exactly
  • Check agent logs for error details
  • Ensure sufficient balance in escrow

Port already in use

Cause: Default port 8500 occupied

Solution:

  • Specify different port in quickConnect: port: 8501
  • Kill existing process: lsof -ti:8500 | xargs kill

Dashboard shows outdated data

Cause: Browser cache

Solution:

  • Hard refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  • Clear browser cache
  • Restart relay server
